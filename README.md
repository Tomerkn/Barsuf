# מערכת Barsuf - מדריך ניהול פרויקטים

המערכת של ברסוף נועדה לנהל את כל היבטי הפרויקטים של החברה במקום אחד מסודר. היא מטפלת במעקב פרויקטים, ניהול תקציב, הכנסות, הוצאות, יומני עבודה ושימוש בבינה מלאכותית לניתוח מסמכים.

### מה המערכת עושה
המערכת מאפשרת לעקוב אחרי ההתקדמות של כל פרויקט בנייה, לנהל את הקשר עם הקבלנים, לתעד עבודה יומיומית מהשטח ולשלוט בתזרים המזומנים (רווח והפסד). בנוסף, יש את "ברבור" - עוזר חכם שמנתח קבלות ומסמכים ומעדכן את הנתונים באופן אוטומטי.

### טכנולוגיות בשימוש
1. React (Javascript): צד הלקוח, כל מה שרואים בדפדפן.
2. Node.js (Express): צד השרת, המנוע שמעבד את הבקשות.
3. SQLite: מסד הנתונים שבו נשמר כל המידע. הקובץ נמצא בכתובת server/data/barsuf.db.
4. Google Gemini API: הבינה המלאכותית שמשמשת לקריאת מסמכים וצ'אט חכם.

### מבנה הקבצים בפרויקט

#### צד לקוח (תיקיית src)
* App.jsx: הקובץ הראשי שמנהל את הניווט בין הדפים.
* pages/: כל דפי המערכת (פרויקטים, דשבורד, תקציב, הוצאות וכו').
* components/: רכיבי ממשק משותפים (כפתורים, חלוניות, והווידג'ט של ברבור).
* services/api.js: השירות שמקשר בין האתר לשרת.

#### צד שרת (תיקיית server)
* index.js: נקודת הכניסה של השרת.
* db.js: ניהול מסד הנתונים והטבלאות.
* ai.js: לוגיקת החיבור לבינה המלאכותית של גוגל.
* data/: תיקייה לשמירת קבצים שהועלו ומסד הנתונים.

#### תשתיות (תיקיית k8s)
* קבצי הגדרה להרצת המערכת בסביבת ענן (קוברנטיס).

### איך מפעילים את המערכת
1. התקנת ספריות: מריצים את הפקודה npm install בתיקייה הראשית.
2. הגדרת מפתח AI: יש להגדיר את GEMINI_API_KEY בקובץ .env בשרת.
3. הרצה: מריצים את הפקודה npm run dev כדי להפעיל את האתר והשרת יחד.
4. כניסה: פותחים את הדפדפן בכתובת המקומית שמופיעה בטרמינל.
